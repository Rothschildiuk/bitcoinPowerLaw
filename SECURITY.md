# Security Policy

## Supported Versions

Security fixes are applied to the latest state of the `main` branch.

## Reporting a Vulnerability

Please do not open a public issue for security-sensitive problems.

If you believe you have found a vulnerability:

1. Report it privately through GitHub security reporting features if available for this repository.
2. If private reporting is not available, contact the maintainer directly through GitHub before disclosing details publicly.
3. Include a clear description, impact, reproduction steps, affected files or flows, and any suggested mitigation if you have one.

## Response Expectations

The maintainer will review reports in good faith and aim to:

- Confirm whether the issue is reproducible.
- Assess the severity and scope.
- Prepare a fix or mitigation before public disclosure when practical.

Please avoid public disclosure until the issue has been triaged and a fix strategy is agreed.
